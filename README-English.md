# Pangya Fun - Legacy Version (v1.0) ⛳

Server based on Acrisio's (SuperSS Dev) code — rebuilt and adapted in C#.

This repository contains the source code for **Version 1.0** of the **Pangya Fun** project. This code represents the foundation of years of reverse engineering and C# development, focused on faithfully emulating the Japanese server experience.

This entire version was **DEDICATED TO THE COMMUNITY PROJECT**, aiming to strengthen the Pangya preservation community.

> **⚠️ WARNING:**
> **This project is provided as a study base. You are free to modify, adapt, or use it as you wish.**

> This version has been officially discontinued and moved to this repository as a historical archive. The current project now uses a modern architecture (.NET 10, Docker).

---

### 📌 Overview

This project simulates the main components of a PangYa server:

- **LoginServer** – Player authentication.

- **MessengerServer** – Messaging system and friends, guild.

- **GameServer** – Lobby, rooms and matches.

- **RankServer** – Player ranking, top 12 players on the map, etc...

- **AuthServer** – Synchronization between servers, data, sending.

It is compatible with the Japanese client **ProjectG JP version 972.00 or higher**.

## 🛠 Architecture and Components (V1.0)

The server was built on a modular foundation divided into specialized APIs:
| API                        | Main Function                                                                         |
|----------------------------|---------------------------------------------------------------------------------------|
| **PangyaAPI.Network**      | Manages TCP connections, sessions, buffers, sending/receiving, and packet handling.   |
| **PangyaAPI.SQL**          | Database access interface (SQL Server), asynchronous commands and responses.          |
| **PangyaAPI.IFF.JP**       | Handles Japanese client IFF files (items, characters, courses, etc.).                 |
| **PangyaAPI.Utilities**    | Auxiliary tools: Log, enums, `.ini` configuration, encryption, error structure.       |

## 🧩 The GUID Challenge (To make it work)

For the rooms to function correctly, there is a small piece of code that is intentionally **commented out**.

This code is responsible for making the game "understand" which **GUID** generated by the server will be used by the client (player). Without it, the connection between the player and the room is not completed.

**Your only challenge is:** Explore the code, find this GUID, and remove the comment. If you study the Pangya protocol, you will know exactly where it is.

## 🗄 Connectivity and Database

Data persistence uses **MSSQL** through robust connection bridges:

* **Driver:** Uses `odbc32` or `odbc64` (configured via System DSN in Windows).

* **Default Configuration (server.ini / config):**

```ini
DBENGINE    = MSSQL
DBIP        = pangya
DBNAME      = pangya
DBUSER      = pangya
DBPASS      = pangya
DBPORT      = 1433
DBLOG       = 0
```
